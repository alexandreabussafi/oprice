Guia Fácil: Como Atualizar e Consertar o Sistema

Este é um passo a passo simples para você, mesmo sem conhecimento técnico, conseguir enviar as atualizações que fizemos no código para o site que está no ar, ou consertar se algo der errado.

🚀 Parte 1: Como colocar uma novidade no ar (Deploy)

Toda vez que terminarmos de fazer uma alteração no código aqui no seu computador, precisamos avisar a "Nuvem" (Vercel) para ela pegar esse novo código e atualizar o site oficial. Siga a ordem abaixo no terminal do VSCode (a telinha preta embaixo):

Passo 1: Preparar as malas

Digite o comando abaixo e aperte Enter. Ele avisa o sistema para embrulhar todos os arquivos que você modificou hoje.

git add .

Atenção: tem um espaço e um ponto final ali no comando.

Passo 2: Colocar uma etiqueta na mala

Agora você vai colocar um bilhete dizendo o que você fez, para lembrarmos depois.

git commit -m "Coloquei o botao de deletar usuario"

Dica: Escreva sempre dentro das aspas o que você acabou de mudar no sistema.

Passo 3: Enviar pelo correio (Para o Ar!)

Por fim, esse comando pega a sua mala etiquetada e envia para a Vercel. Esse botão "Publica" o site.

git push origin main

Pronto! Agora é só esperar de 1 a 2 minutos e o seu site oficial estará atualizado com as novas funções.

🚑 Parte 2: O botão de pânico (Rollback)

Imagine que você seguiu os 3 passos acima, a atualização foi para o ar, mas o site quebrou ou os clientes não conseguem usar por causa de um erro imprevisto.

A Vercel tem uma máquina do tempo super fácil de usar, onde você aperta um botão e o site volta a ser exatamente a versão anterior que funcionava perfeitamente.

Como usar a máquina do tempo no site Vercel (Jeito mais fácil)

  1. Abra o seu navegador de internet e entre no site vercel.com.
  2. Faça login (geralmente com o seu GitHub ou e-mail).
  3. Clique no quadradinho do seu projeto (chamado oprice ou opcapex).
  4. Na barra que fica no topo, clique na palavra Deployments (significa Envios).
  5. Você vai ver uma lista com todos os envios que você já fez na vida, com as datas. O primeiro lá em cima (o mais recente) é o seu envio com erro.
  6. Olhe para o envio logo abaixo dele (que era o de ontem ou do dia que funcionava).
  7. No cantinho direito da linha desse envio antigo, clique nos Três Pontinhos (•••).
  8. No menu que abrir, clique na opção chamada Promote to Production (algumas vezes pode aparecer escrito apenas Rollback).
  9. Vai aparecer uma janelinha pedindo confirmação. Clique no botão de confirmar.

UFA! Em 2 segundos o seu site tira a versão "quebrada" do ar e coloca a versão "antiga que funciona" de volta. Isso é excelente para não deixar o cliente na mão enquanto você e eu voltamos aqui no VSCode para consertar o código com calma.
